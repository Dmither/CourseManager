# Проєкт: «Платформа керування навчальними курсами» (CourseManager) 

Коротко: реалізувати повноцінну систему керування курсами, студентами, 
викладачами, модулями курсу, оцінками, розкладом і плагінами для 
експорту/імпорту. Проєкт має показувати застосування всіх парадигм і принципів 
ООП у Python. 

## Основні вимоги 

1. Моделі домену (класи)
- Course, Module, Lesson, User (абстрактний), Student, Instructor, 
Admin.
- Enrollment (зв’язок студент↔курс), Grade, Assignment, ScheduleEvent.
- Використати наслідування (User → Student/Instructor/Admin), 
композицію (Course містить Module(s)), агрегацію.
2. Інтерфейси/абстракції
3. Принципи SOLID
- Показати SRP: кожен клас — одна відповідальність.
- OCP: можливість додавати нові плагіни/стратегії без зміни існуючого 
коду.
- LSP: підкласи можуть замінювати базові класи без порушення 
поведінки.
- ISP: розбиття великих інтерфейсів на дрібні.
- DIP: залежності мають інжектитись через конструктор або фабрику.
4. Патерни проєктування. Можна по бажанню.
5. Магічні методи
- Перевантажити __repr__, __str__, __eq__, __len__, __iter__ там, де логічно.
6. Серіалізація / Збереження
- Зберігати/завантажувати курси та користувачів у JSON або SQLite.
- Плагіни імпорту/експорту (CSV, JSON).
7. Код-стайл
- Виконати PEP8.
8.  Збережння проекту: 
- В PyCharm. 
- Git та GitHub.
